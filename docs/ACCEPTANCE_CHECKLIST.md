# 内容创作与自动分发 Agent 可验收清单

## 验收范围

本阶段以“公众号内容创作闭环 MVP”为主线，目标是确认系统可以稳定完成：

- 输入创作需求
- 生成公众号文章
- 编辑与保存内容
- 生成或选择公众号首图
- 导出内容
- 发布到公众号草稿箱
- 保留历史记录、草稿和内容库

小红书、Twitter、视频脚本在本阶段作为“生成、编辑、导出、保存”能力验收，真实发布不纳入当前闭环。

## 一、基础访问与页面结构

- [ ] 首页可以正常打开，无运行时报错。
- [ ] 首页能看到创作输入框、平台选择、草稿箱、内容库预览和设置入口。
- [ ] 工作台可以正常打开，左侧历史记录、右侧平台内容区布局稳定。
- [ ] 设置页可以正常打开，Skills Library、平台规则、技能上传/安装入口可见。
- [ ] 页面在浏览器刷新后不丢失已生成的历史记录。

## 二、公众号创作闭环

- [ ] 用户输入需求并选择“公众号文章”后，可以生成一篇结构完整的公众号文章。
- [ ] 生成结果包含标题、摘要、正文。
- [ ] 正文支持 Markdown/富文本结构的基础呈现，包括标题、加粗、引用、分隔线、图片占位或图片链接。
- [ ] 点击“编辑”后可以修改标题、摘要和正文。
- [ ] 修改后返回、刷新或切换历史记录时，内容不会丢失。
- [ ] 点击“复制”可以复制公众号正文内容。
- [ ] 点击“导出 Markdown”可以下载 Markdown 文件。
- [ ] 点击“导出 HTML”可以下载 HTML 文件。

## 三、公众号首图

- [ ] 公众号文章工作台中存在“公众号首图生成”版块。
- [ ] 点击生成首图后可以生成一张可预览的封面图。
- [ ] 首图不应出现与文章无关的平台 Logo、社交图标、手机 UI 框、WeChat/Instagram 等无关文字。
- [ ] 首图应尽量减少图片内文字，优先使用主题相关视觉、人物、场景、抽象意象或插画表达。
- [ ] 首图生成失败时应显示可读错误提示，而不是静默失败。
- [ ] 生成后的首图可以用于公众号发布的封面图。

## 四、公众号发布到草稿箱

- [ ] 点击“发布到公众号”后弹出发布管理弹窗。
- [ ] 弹窗可以拉取并展示当前 API Key 下的公众号列表。
- [ ] 用户必须先选择公众号账号。
- [ ] 用户可以选择发布类型：公众号文章或小绿书图文。
- [ ] 未选择公众号时不能发布，并显示明确提示。
- [ ] 发布普通公众号文章时，内容发布到公众号草稿箱，不会直接群发。
- [ ] 发布成功后页面显示醒目的“成功保存到草稿箱”提示。
- [ ] 发布失败时显示明确错误原因，例如 API Key 无效、账号未授权、授权过期、参数错误、微信接口错误。
- [ ] 发布到公众号后台后，Markdown 中的标题、二级标题、加粗、引用等基础格式应尽量保持对应的 HTML 呈现。

## 五、历史记录、草稿箱与内容库

- [ ] 每次生成内容后会创建历史记录。
- [ ] 左侧历史记录支持点击进入对应创作工作台。
- [ ] 历史记录支持搜索、重命名和删除。
- [ ] 首页需求草稿箱可以保留多条需求草稿。
- [ ] 草稿箱支持继续编辑、删除。
- [ ] 已生成的公众号文章可以加入内容库。
- [ ] 内容库预览展示标题和摘要。
- [ ] 点击内容库文章可以查看正文并继续编辑。
- [ ] 草稿箱和内容库列表有滚动区域，不会无限撑高首页。

## 六、Skills 与提示词配置

- [ ] 设置页可以上传 ZIP 技能包。
- [ ] 技能包至少能解析 `SKILL.md`。
- [ ] 技能详情可以查看技能摘要、文件列表和文件内容预览。
- [ ] 可以通过 GitHub 安装技能。
- [ ] 技能库中的技能支持删除。
- [ ] 平台规则中可以为空，也可以选择已安装技能。
- [ ] 平台规则支持保存和重置。
- [ ] 公众号平台可以选择内容技能或提示词技能参与生成。

## 七、小红书笔记，本阶段只验收生成与导出

- [ ] 选择“小红书笔记”后可以生成标题、文案、标签和配图建议。
- [ ] 小红书 AI 生图默认关闭，避免测试阶段消耗过多 API token。
- [ ] 用户打开生图开关并重新生成后，才会调用图片模型生成配图。
- [ ] 配图区域支持预览、放大、保存单张、批量保存。
- [ ] 图片生成失败时支持单张重试和批量重试。
- [ ] 小红书内容可以导出 Markdown、HTML 和图片包。
- [ ] 本阶段不验收真实发布到小红书账号。
- [ ] 小红书发布按钮如保留，应明确标注为“创建发布链接/待手机端确认”，不得误导为已直接发布到账号。

## 八、Twitter，本阶段只验收生成与编辑

- [ ] 选择 Twitter 后可以生成 Single 或 Thread 内容。
- [ ] Auto、Single、Thread 模式切换应有明确解释。
- [ ] Twitter 默认生成语言为 English，用户可手动指定语言。
- [ ] Single Tweet 不应超过 280 字符。
- [ ] Thread 每条推文应独立可编辑。
- [ ] 点击 Twitter 版块的“重新生成”只重新生成 Twitter 内容。
- [ ] 本阶段不验收真实发布到 Twitter。

## 九、视频脚本，本阶段只验收生成与导出

- [ ] 选择“视频脚本”后可以生成结构化脚本。
- [ ] 脚本以表格形式展示。
- [ ] 表格字段包括：镜号、文案内容、画面建议、字幕重点、节奏、音效/音乐、特效。
- [ ] 表格内容完整展示，必要时只保留横向滚动，不应在单元格内部出现影响阅读的滚动条。
- [ ] 点击复制应复制可给视频团队使用的表格内容，而不是 JSON 代码格式。
- [ ] 支持导出视频脚本文档。

## 十、错误提示与稳定性

- [ ] 模型超时、余额不足、搜索失败、图片生成失败、发布失败都有明确提示。
- [ ] 失败后用户能知道下一步该怎么处理，例如重试、换图、检查 API Key、重新生成。
- [ ] 刷新页面后不应出现 Webpack runtime 崩溃或模块找不到错误。
- [ ] 控制台不应出现影响主流程的 500/503/运行时异常。
- [ ] 常用操作连续执行后页面不应卡死，例如多次重新生成、切换平台、保存编辑、打开历史。

## 十一、阶段验收结论

验收通过条件：

- [ ] 公众号创作到草稿箱发布链路完整通过。
- [ ] 历史记录、草稿箱、内容库可正常沉淀内容。
- [ ] 设置页 Skills 基础能力可用。
- [ ] 小红书、Twitter、视频脚本至少满足生成、编辑、导出，不要求真实发布。
- [ ] 关键错误提示可读，用户知道失败原因和下一步。

暂不纳入本阶段验收：

- [ ] 小红书真实账号选择与最终发布。
- [ ] Twitter 真实发布。
- [ ] 多账号登录体系。
- [ ] 复杂公众号模板排版。
- [ ] 图片资产长期云存储、配额、清理策略。
- [ ] 内容版本回滚系统。
