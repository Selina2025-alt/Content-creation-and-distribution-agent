# 项目进度总览

## 项目名称
内容创作与自动分发 Agent

## 项目目标
基于 Next.js + SQLite 构建多平台内容工厂，支持公众号、小红书、Twitter、视频脚本的生成、编辑、导出、技能驱动与发布管理，并持续提升内容质量与发布可用性。

## 当前整体进度
92%（MVP Alpha 收尾：功能链路基本打通，持续做稳定性与体验优化）

## 已完成内容
- 首页创作入口、平台多选、工作台、历史记录、草稿箱、内容库主流程已打通。
- 多平台内容生成链路已接入（公众号 / 小红书 / Twitter / 视频脚本）。
- Skills Library 已支持 ZIP 上传、GitHub 安装、Prompt 技能、技能摘要预览、删除、平台绑定。
- 小红书图片链路已支持生成、失败重试、单张保存、批量保存、批量重试与发布前图片可访问性检查。
- 公众号发布已接入真实接口（公众号列表 + 发布到草稿箱），并支持发布类型选择（文章 / 小绿书）。
- 小红书发布已接入真实接口，发布后返回 `publish_url` 并在前端弹窗展示二维码。
- 导出能力已覆盖 Markdown、HTML、图片包、视频脚本文档。
- 本阶段新增：小红书生图开关（默认关闭）已贯通首页创建、工作台重新生成、后端任务接口与生成服务。

## 当前正在进行
- 公众号首图生成效果优化（降低无关元素与脏文本，提升风格稳定性）。
- 发布失败错误文案的可读性优化与分级提示。
- Twitter 发布能力的后续真实接口接入准备。

## 待完成事项
- 接入 Twitter 真实发布 API。
- 统一模型配置中心（文本模型、图片模型、搜索策略、生成参数）。
- 完善内容版本管理（生成版/编辑版/发布版与回滚）。
- 图片资产生命周期治理（容量、过期、清理策略）。

## 已解决的问题
- 修复工作台初始化时 `imageAssets` 为空导致崩溃（`.some` 空值问题）。
- 修复首页测试文件字符串损坏导致的语法错误，重建可维护测试用例。
- 修复小红书发布前图片过期识别与提示（支持 `XIAOHONGSHU_IMAGE_EXPIRED` 反馈）。
- 修复小红书发布链路 API Key 配置问题，支持独立环境变量读取。

## 当前存在的问题 / 风险 / 卡点
- 图片模型对“绝对无文字/无图标”约束仍有随机性波动。
- 第三方图像服务在高并发时会出现耗时与失败率波动。
- 小红书发布强依赖公网可访问图片 URL，本地占位图仍需先转换为公网图。

## 下一步计划
- 完成小红书“默认不生图”开关在工作台的交互细节优化（文案提示、状态回显）。
- 增加对应 API/页面回归测试（创建任务 + 单平台重生 + 历史偏好继承）。
- 继续推进公众号首图效果优化与异常回退策略。
