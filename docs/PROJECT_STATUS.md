# 项目进度总览

## 项目名称
内容创作与自动分发 Agent

## 项目目标
构建一个基于 Next.js + SQLite 的多平台内容工厂，支持内容生成、编辑、技能扩展、导出和发布闭环，并可被团队成员快速安装与复用。

## 当前整体进度
96%（MVP 功能闭环完成，进入稳定性和效果优化阶段）

## 已完成内容
- 已完成首页创作入口、平台多选、工作台、历史记录、草稿箱、内容库主流程。
- 已完成公众号、小红书、Twitter、视频脚本四平台生成和编辑链路。
- 已完成 Skills Library（ZIP 上传、GitHub 安装、Prompt 技能、绑定、删除、摘要预览）。
- 已完成导出能力（Markdown、HTML、图片包、视频脚本文档）。
- 已接入公众号发布 OpenAPI（账号列表 + 发布到草稿箱）。
- 已接入小红书发布 OpenAPI（创建发布任务并返回发布链接/二维码）。
- 已完成小红书图片批量任务管理（重试、并发、保存等）。
- 已完成“默认不生图”控制开关，降低测试阶段 token 消耗。

## 当前正在进行
- 公众号首图生成效果优化（去除无关图标、伪 UI 元素、过多文本）。
- 发布反馈文案优化（错误更可读、状态更清晰）。

## 待完成事项
- 接入 Twitter 真实发布 API。
- 统一模型配置中心（文本、图像、搜索策略、参数模板）。
- 完善版本管理（生成版/编辑版/发布版对比与回滚）。
- 完成图片资产生命周期策略（过期、配额、清理）。

## 已解决的问题
- 修复 `imageAssets` 空值导致工作台初始化报错。
- 修复小红书发布链路 API Key 读取优先级问题。
- 修复小红书发布前图片 403 过期识别和重试入口。
- 修复“创建发布链接 ≠ 直接代发账号”的用户理解歧义（增加确认与说明）。

## 当前存在的问题 / 风险 / 卡点
- 图像模型偶发生成无关角标、图标和 UI 痕迹，仍需持续压制。
- 第三方图像服务存在高峰期延迟和失败波动。
- 小红书发布依赖公网可访问图片 URL，本地临时图需先转可访问地址。

## 下一步计划
- 优先优化公众号首图生成质量，继续降低“角标/图标/手机框”残留率。
- 增加发布前可视化校验清单（封面可访问性、图片数量、内容长度）。
- 补齐发布链路端到端回归测试（公众号、小红书扫码发布）。
