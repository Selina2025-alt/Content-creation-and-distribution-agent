# 项目进度总览

## 项目名称
内容创作与自动分发 Agent

## 项目目标
基于 Next.js + SQLite 构建多平台内容工厂，支持公众号文章、小红书笔记、Twitter、视频脚本的生成、编辑、导出、技能扩展与发布管理，并持续优化质量与发布可用性。

## 当前整体进度
95%（MVP 闭环完成，进入稳定性与质量优化阶段）

## 已完成内容
- 已完成首页创作入口、平台多选、工作台、历史记录、草稿箱、内容库主流程。
- 已完成四平台内容生成与编辑器（公众号 / 小红书 / Twitter / 视频脚本）。
- 已完成 Skills Library：ZIP 上传、GitHub 安装、Prompt 技能、摘要预览、删除、平台绑定。
- 已完成小红书图片能力：单张保存、批量保存、失败重试、并发控制、任务耗时统计。
- 已完成导出能力：Markdown、HTML、图片包、视频脚本文档。
- 已接入公众号发布 OpenAPI（账号列表 + 发布到草稿箱，含错误码提示）。
- 已接入小红书发布 OpenAPI（创建发布任务并返回 publish_url / 二维码）。
- 已完成发布前图片可访问性检查（403 过期图识别）与一键重生图修复。
- 已完成“生成内容默认不生图”开关，降低测试阶段 token 消耗。

## 当前正在进行
- 公众号首图生成质量优化（去除无关 logo / UI 元素 / 伪社媒角标）。
- 发布状态提示与错误文案精细化（让业务状态更清晰可读）。

## 待完成事项
- 接入 Twitter 真实发布 API。
- 统一模型配置中心（文本模型、图像模型、搜索策略、参数模板）。
- 完善版本管理（生成版/编辑版/发布版对比与回滚）。
- 完成图片资产生命周期治理（配额、清理策略、过期策略）。

## 已解决的问题
- 修复 `imageAssets` 空值导致的工作台初始化崩溃。
- 修复小红书发布链路 API Key 读取优先级问题。
- 修复小红书发布前图片过期（403）无引导的问题，新增结构化错误与重试入口。
- 修复发布链路中“创建发布链接”和“最终账号侧发布”理解歧义，新增确认弹窗与说明。

## 当前存在的问题 / 风险 / 卡点
- 图像模型仍可能偶发生成无关角标、伪 UI 元素，需要持续负向约束与后处理。
- 第三方图像接口在高峰时段存在延迟和失败波动。
- 小红书发布仍依赖公网可访问图片 URL，本地临时图需先转换后发布。

## 下一步计划
- 优先持续优化公众号首图效果，重点压低“角标/图标/手机框”残留率。
- 增加发布前可视化检查清单（封面可访问性、图片数量、内容长度、必填项）。
- 补齐发布链路端到端回归测试，覆盖公众号发布与小红书二维码发布流程。
