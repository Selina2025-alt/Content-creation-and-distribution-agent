# 项目进度总览

## 项目名称
内容创作与自动分发 Agent

## 项目目标
基于 Next.js + SQLite 搭建多平台内容工厂，支持创作、编辑、技能扩展、导出与发布闭环，并可被他人一键安装和快速上手。

## 当前整体进度
97%（MVP 主流程闭环完成，进入稳定性与体验优化阶段）

## 已完成内容
- 首页创作入口、草稿箱、历史记录、内容库、工作台主流程已完成。
- 公众号/小红书/Twitter/视频脚本四平台生成与编辑链路已打通。
- Skills Library 已支持 ZIP 上传、GitHub 安装、Prompt 技能、摘要预览、删除与平台绑定。
- 导出能力已支持 Markdown、HTML、图片包、视频脚本文档。
- 公众号发布已接入真实 OpenAPI（账号列表 + 发布到草稿箱）。
- 小红书发布已接入真实 OpenAPI（创建发布任务 + 返回发布链接/二维码）。
- 小红书配图批量任务管理已支持失败重试、批量保存、并发控制等基础能力。
- 独立仓库分发版 README、.env 模板与安装说明已补齐。

## 当前正在进行
- 公众号首图生成质量优化（减少无关图标、UI 痕迹、脏文字）。
- 发布失败提示文案可读性优化（更清晰区分参数问题、鉴权问题、资源不可访问问题）。

## 待完成事项
- Twitter 真实发布 API 接入。
- 模型配置中心统一化（文本模型、图片模型、搜索策略、参数模板）。
- 内容版本管理完善（生成版/编辑版/发布版对比与回滚）。
- 图片资产生命周期治理（配额、过期、自动清理策略）。

## 已解决的问题
- 修复工作台初始化时 `imageAssets` 为空导致报错。
- 修复小红书发布前图片过期（403）识别与重试流程。
- 修复小红书发布 API Key 读取优先级导致的“密钥无效”问题。
- 修复仓库分发时远端已有初始化提交导致的推送冲突。

## 当前存在的问题 / 风险 / 卡点
- 图片模型在少数样本上仍会生成与文案无关的角标/图标元素。
- 第三方图片服务高峰期会有耗时抖动和失败率波动。
- 小红书发布依赖公网可访问图片 URL，本地临时图仍需先转换为公网资源。

## 下一步计划
- 继续压制公众号首图中的无关视觉元素，提升可用率。
- 增加发布前可视化校验清单（封面可访问性、图片数量、内容长度）。
- 补齐端到端回归测试：发布成功、失败重试、历史恢复、导出一致性。
